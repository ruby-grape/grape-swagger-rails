Upgrading Grape Swagger Rails
=============================

### Upgrading to 1.0.0

Version `1.0.0` includes a major Swagger UI integration refresh and several configuration additions.

#### Upgrade checklist

1. Verify your Ruby/Rails/Grape/`grape-swagger` versions against the compatibility matrix in `README.md`.
2. The template was converted from ERB to Haml. If you override the engine template, the new path is `app/views/grape_swagger_rails/application/index.html.haml` (previously `index.html.erb`).
3. If you override Swagger page JavaScript behavior, rebase customizations on:

   - `app/assets/javascripts/grape_swagger_rails/index.js`

   The UI runtime is now maintained from TypeScript source and compiled into this asset.

4. **Request body encoding has changed for POST/PUT/PATCH endpoints.**

   Swagger UI v5 correctly renders `in: body` parameters (as generated by grape-swagger) as a single JSON body editor, sending `Content-Type: application/json`. Swagger UI 2.x rendered those same parameters as individual form fields and sent `Content-Type: application/x-www-form-urlencoded`. No spec changes are needed if your API accepts JSON.

   If your endpoints explicitly consume `application/x-www-form-urlencoded` or `multipart/form-data`, declare your parameters as `in: formData` in your Grape API definition to restore the per-field UI.

### Upgrading to >= 0.2.3

#### Changes to Options

The `GrapeSwaggerRails.options.before_filter` option have been deprecated since 0.2.3.

Please use `GrapeSwaggerRails.options.before_action` instead.
